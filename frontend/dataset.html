Dataset and implementation

How WeatherML collects data and trains the model.

The dataset is not manually downloaded from Kaggle. WeatherML builds it from live Open-Meteo historical archive data, converts it into ML rows, evaluates models, and publishes review artifacts.

Training command

python3 backend/aiml/training_pipeline.py --city Hyderabad --days 730

Change the city and days to rebuild the dataset for another location.

01

Resolve location

Use the geocoding API to convert a city name into latitude, longitude, timezone, and region metadata.

02

Fetch archive data

Pull daily temperature, precipitation, wind, gust, and weather-code history from Open-Meteo.

03

Build ML rows

Use yesterday's weather plus seasonal sine/cosine signals to predict the next day's maximum temperature.

04

Publish artifacts

Write metrics, predictions, feature weights, and model card files consumed by the app and README.

How to rebuild

Change city or window

python3 backend/aiml/training_pipeline.py --city Visakhapatnam --days 1095

This fetches a new historical window, regenerates the model registry, and refreshes frontend artifacts.
